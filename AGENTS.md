# Codex 작업 규칙

- 모든 구현은 모바일 우선 반응형 웹 기준으로 한다.
- 초기 MVP는 단순하고 안정적인 구조를 우선한다.
- 새 기능을 추가하기 전에 docs 문서를 먼저 확인한다.
- 한 번에 큰 범위를 구현하지 말고 작은 PR 단위로 작업한다.
- 임의로 MVP 범위를 확장하지 않는다.
- 인테리어 업체 연결, 부동산 중개, 실시간 채팅, 결제, 앱 기능은 구현하지 않는다.
- 변경 후에는 README/docs 링크가 깨지지 않는지 확인한다.
- TypeScript를 사용하는 경우 any 남발을 피하고 명확한 타입을 둔다.
- 화면 문구는 고객에게 과장된 단가 비교처럼 보이지 않게 한다.
- 핵심 문구는 “창업 조건을 정리해서 납품 상담을 제대로 받을 수 있게 도와주는 곳”을 따른다.
